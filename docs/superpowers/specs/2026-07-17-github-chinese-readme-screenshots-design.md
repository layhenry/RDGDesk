# GitHub 中文首页与界面截图设计

## 目标

让 RDGDesk 的 GitHub 仓库默认以简体中文介绍项目，同时保留内容完整、便于海外用户阅读的英文版本。仓库首页需要展示真实软件界面，让访问者能够快速理解资源库、远程连接与凭据设置等核心能力。

本次只调整仓库文档、截图资源和 GitHub 项目简介，不修改应用功能、数据格式、许可证条款或运行行为。

## 语言结构

- `README.md` 作为默认简体中文首页。
- `README.en.md` 保存完整英文版本。
- 两份 README 顶部都提供显眼的语言切换链接：`简体中文 | English`。
- GitHub 仓库 Description 使用中文，保留 `macOS`、`RDCMan`、`.rdg` 和 `FreeRDP` 等必要技术名称。
- `LICENSE` 保持英文原文，避免改变许可证法律文本。
- `TRADEMARKS.md` 与 `THIRD_PARTY_NOTICES.md` 保持现有英文权威文本；中文 README 提供准确的中文摘要并链接原文。

## README 信息架构

中文首页按以下顺序组织：

1. 项目名称、语言切换和一句话定位。
2. 主界面大图。
3. 安全与开发状态提醒。
4. 核心功能。
5. 界面预览图组。
6. 安装依赖、运行、打包与开发验证命令。
7. 安全模型、兼容性与商标说明。
8. 许可证和第三方软件说明。

英文页面沿用相同结构和图片，避免中英文版本功能信息不一致。

## 截图方案

采用真实软件截图，不使用与当前实现不一致的生成式假界面。图片存放在 `docs/images/`，使用适合 GitHub 展示的 PNG，并在中英文 README 中共用。

计划包含四张图：

1. `rdgdesk-main.png`：资源库与远程桌面主界面，作为首页主图。
2. `rdgdesk-settings.png`：设置首页或通用设置，体现原生 macOS 风格。
3. `rdgdesk-credentials.png`：全局凭据和继承范围设置，密码必须完全不可见。
4. `rdgdesk-connection.png`：成功连接或连接诊断界面，展示实际远程能力和清晰错误信息。

截图应保持一致的窗口尺寸、浅色外观和原生 macOS 视觉。主图使用较宽画幅；其余三张以表格方式并排或分组展示，窄屏下允许自然换行。

## 数据脱敏规则

- 不使用用户的真实 `.rdg` 文件、服务器名称、IP、端口、用户名、证书指纹或连接日志。
- 截图仅使用虚构分组与服务器，例如“演示环境”“测试服务器”。
- 地址只使用 `rdp.example.test` 或 RFC 5737 文档示例网段，例如 `198.51.100.57`。
- 所有密码字段保持空白或系统掩码，不显示长度可推断的真实密码。
- 截图前关闭与项目无关的桌面窗口、通知和个人账户信息。
- 导出后再次进行 OCR/文字检查，确认图片中没有真实服务器或个人资料。

## 实施边界

- 优先通过应用自身界面创建虚构演示数据并截取真实窗口。
- 如果某个功能页当前无法在不接触真实数据的情况下展示，则跳过该图，不使用用户数据补位。
- 不在 GitHub 上传 `.rdg` 演示库；仓库继续只保留现有虚构测试夹具。
- 不上传安装包、Keychain 内容或运行日志。

## 验证

- 检查 `README.md` 默认中文且所有内部链接有效。
- 检查 `README.en.md` 内容完整，并可从中文页面一键切换。
- 检查四张图片可以在 GitHub Markdown 中正常显示，文件大小适合网页加载。
- 使用文本搜索和图片 OCR 检查真实服务器特征、用户名、密码和 `temp2.rdg` 均未出现。
- 运行文档链接检查、`git diff --check` 和现有快速测试，确认文档更改没有破坏项目。
- 推送后通过 GitHub API 和公开仓库页面核对默认 README、中文 Description、远端提交及图片资源。

## 完成标准

访问 `layhenry/RDGDesk` 时默认看到中文首页和真实脱敏界面图；英文用户可在首页顶部进入完整英文说明；公开仓库中不出现任何用户真实远程桌面数据。
